# Proof Length Mismatch Error

## Error Description

When running tests, the verifier contract fails with the following error:

```
Error: VM Exception while processing transaction: reverted with custom error 'ProofLengthWrongWithLogN(12, 6624, 7488)'
```

This error occurs at `HonkVerifier.verify()` in `contracts/Verifier.sol:1939`.

## Error Details

- **Expected proof size**: 7488 bytes (234 fields × 32 bytes)
- **Actual proof size**: 6624 bytes (207 fields × 32 bytes)
- **Difference**: 864 bytes (27 fields)
- **LOG_N**: 12 (circuit size 4096)

## Root Cause

The verifier contract expects a proof of size 7488 bytes based on `LOG_N = 12`, but the proof generated by `UltraHonkBackend.generateProof()` is only 6624 bytes. This indicates a mismatch between:

1. The circuit size used to generate the verifier contract
2. The actual circuit size of the compiled Noir circuit

## Additional Information

### Version Compatibility

The following tools are used in the verification process:

- **`bb` CLI tool**: Used to generate the Solidity verifier contract via `bb write_solidity_verifier`
- **`@aztec/bb.js` package**: Used to generate proofs via `UltraHonkBackend.generateProof()`
